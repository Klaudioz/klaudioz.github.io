My personal knowledge management system

Klaudioz's blog
2020-11-12
Knowledge management, Tools
8 min. read

Everybody is trying to find or create the one true note-taking tool or knowledge management system. Unfortunately, the complexity of any of these tools makes this task impossible to accomplish.
On the contrary, there are hundreds of available tools, and make the correct integration between them is the real challenge. But being honest, people don’t have time for that and everybody wants a simple solution which the knowledge management market probably isn’t mature yet for that.
I’ve used a lot of these tools and I can speak about my experience but for me, the final workflow is still WIP and probably it’ll be in that way forever.
So, I’m going to describe some tool categories, which any of these have maybe dozens of tools available so I’m going to describe only the tools I have experience working with them or I have some familiarity with.
My main idea for this is to use a Zettelkasten and try to centralize everything.
I use multiple tools and techniques to make this workflow ideally frictionless and making the correct relationships between the things I’m interested to learn because that’s the only way to don’t forget them. This is the reason I started to research Zettelkasten and finally adopted it, but it can be a new entire topic to discuss in another blog entry.

  • Note-taking & note-management

    • This is a huge world where the most popular tools are: Evernote, Microsoft OneNote, Google Keep, Apple Notes, Roam Research, etc. Even a simple notepad can be used for that. What I’m looking for here is to use Zettelkasten to centralize my knowledge and get compound interest of my thoughts, so to accomplish that I need to make associations between notes. This system model in a better way how the human brain works. The opposite of that and maybe a very common situation a few years ago is to use a tool like Evernote and store and store more data using a hierarchy or tree, something which is excellent for machines but not natural for the human brain so at the end our notes finish its life in a cemetery of thoughts.
    • Another important thing for me is to be able to access my Zettelkasten offline and online on all my devices. Right now in my MacBook Pro, I’m using Obsidian. It’s a great project with constant new features and a great community. Also, all the notes are simple markdown files so I can change it to another tool in the future easily if it’s needed.
    • All my notes are hosted over Dropbox, where I have a lot of space and I can use it for free only using three devices, so I’m using it over my MacBook, Ipad, and iPhone. So most of the time I take notes over the MacBook Pro with Obsidian and if I’m using the iPhone or Ipad I use 1Writer because Obsidian doesn’t have a mobile version yet so everything is coordinated between devices.

  • Quick capture

    • This is extremely important because a thought, a link between ideas, etc is something which can be lost forever easily if it’s not captured fast so I’m using Raindrop on all my devices to capture any URL, tweet, Reddit, email or Youtube video easily managing a collection of bookmarks which should be consumed. It works on all devices and the non-Pro version is free. Right now I’m planning to move to the premium version, but until now I don’t need it at all.
    • I used a few categories for that, the most important are: Articles, Twitter, Reddit (There is an issue where I can’t see the website directly from the app), Newsletters, and Youtube. I used several IFTTT workflows to send to Raindrop anything I like to analyze with more detail so after like a tweet or Youtube video or save a post in Reddit, the IFTTT workflow will create new entries for me in Raindrop. The easiest case of capture is when I’m using a browser, I can send the link to Raindrop sharing with my iPhone/Ipad or using the Raindrop Chrome extension.
    • When finally I process any of these bookmarks, after taking my notes in the Zettelkasten, I move the bookmark to the Done category so I don’t need to use any tag or highlight with this workflow and this is the main reason I stopped to use Instapaper. My end goal using this app is to keep Raindrop with zero bookmarks over all the categories, except Done. That’s very important to me to keep this workflow running smoothly and I try to process all the new bookmarks every morning when I’m more energized after a good night of sleep.
    • All this process can be done with tools different than Obsidian, like using plugins for Visual Studio Code like Foam or Dendron

  • Books

    • This workflow is a simple one. I read mainly on my Ipad using Apple Books, but I can do it on my iPhone and my MacBook preserving highlights. I try to read only one book simultaneously while trying to finish it soon while I’m taking highlights and some notes. When I finish the book, I use ibooks2markdown to get the highlights and copy and paste the generated markdown over a new note in Obsidian. I read every highlight and try to make connections over Obsidian using bi-directional linking. The reason to read the book faster is I’m going to start forgetting the notes and highlights if the time is running out. Also probably there much good information from the book, so the process the analyze and connect any highlight can take some time.

  • Version Control

    • My Zettelkasten is stored in Dropbox so it’s very secure there. But it’s very important to me, so I used an Obsidian plugin called obsidian-git to have an updated copy in GitHub as well. It can be used any cloud storage for that, like Google Drive, Icloud, etc and any version control like GitLab, BitBucket, etc.

  • Visual thinking

    • My workflow right now isn’t using any visual thinking tool, but in a different context I’ve used applications like Muse or Miro which are great but a little bit expensive. If I have to use some visual resources to organize my thoughts, I prefer to use Notability using the Apple Pencil over the Ipad and draw some diagrams.
